## 1. 项目概述

### 1.1 项目背景

客户计划打造一个面向全球高端安防市场的 B2B 品牌官网，主推两大核心解决方案：**要地防卫（Area Defense）** 和 **空天地一体化巡检**。网站核心目标为获取高质量询盘（Leads），而非直接线上成交。这是一个出海项目，面向中东、非洲、北美、亚太、欧洲等市场。

### 1.2 项目目标

- 快速上线一个专业的 B2B 安防解决方案官网
- 建立品牌可信度，触达目标决策者
- 获取中东、非洲、北美目标客户的询盘（RFQ）
- 支持英文 + 阿拉伯语（双语），后期可扩展

### 1.3 项目范围

**包含内容**
- 响应式官网（桌面 + 移动端）
- 两大产品线的内容展示
- 4类目标行业的解决方案页面
- 案例展示（地图 + 项目列表）
- 询盘表单（Contact / RFQ）
- SEO 基础优化
- 多语言支持（英文 + 阿拉伯语 RTL）

**不包含内容（本期 MVP）**
- 客户后台 / 登录系统
- 在线支付 / 电商功能
- 实时聊天机器人
- 复杂 CMS（内容管理由代码层控制）
- 数据可视化或监控平台接入

## 5. 非功能需求

### 5.1 性能

| 指标 | 目标 |
|------|------|
| Lighthouse Performance Score | ≥ 90 |
| 首屏加载时间（FCP） | ≤ 2.5s |
| LCP（Largest Contentful Paint） | ≤ 4.0s |
| 页面总大小（未压缩） | ≤ 3MB |
| CDN 覆盖 | 中东（迪拜）+ 北美 + 欧洲边缘节点 |

### 5.2 安全

- 全站 HTTPS（Let's Encrypt 或 Cloudflare SSL）
- 表单防机器人（CAPTCHA 或 Honeypot）
- CSP（Content Security Policy） headers
- 询盘数据不暴露在 URL 参数中

### 5.3 SEO

| 项目 | 要求 |
|------|------|
| Meta Title | 每个页面独立设置，含关键词 |
| Meta Description | 每个页面独立设置，150-160字符 |
| Open Graph Tags | 首页 + 主要产品页 |
| Structured Data | Organization、Product、FAQPage Schema |
| Sitemap | 自动生成，提交 Search Console |
| Robots.txt | 允许抓取，禁止低价值页面 |
| Canonical URL | 全站统一，防止重复内容 |
| Hreflang | en + ar 互指，x-default |
| 图片 Alt Text | 所有图片有描述性 alt |
| 页面加载速度 | ≥ 90（Lighthouse） |

### 5.4 合规

- 阿拉伯语 RTL 布局正确（文字方向、表格、导航镜像）
- 隐私政策符合 GDPR（欧洲访客）+ 中东当地法规
- 询盘表单包含数据使用声明和同意勾选
- 无 Cookie 弹窗（建议使用 Plausible/Umami 等无 Cookie 分析）

## 6. 技术栈建议

### 6.1 技术选型

| 层级 | 推荐方案 | 理由 |
|------|---------|------|
| **前端框架** | Next.js 14（App Router） | SSR/SSG 兼顾，SEO 友好，React 生态 |
| **样式** | Tailwind CSS | 快速开发，支持 RTL，CDN 友好 |
| **国际化** | next-intl | 成熟、SEO 友好、RTL 支持好 |
| **部署平台** | Vercel | Next.js 官方，全球 CDN，免费套餐够用 |
| **CDN / DNS** | Cloudflare | 免费的 CDN + 抗 DDoS + 边缘缓存 |
| **表单处理** | Tally（或自建 Next.js API Route） | 5 分钟接入，无代码，邮件通知 |
| **分析统计** | Plausible Analytics | 无 Cookie，合规，轻量 |
| **邮件** | Resend | 开发友好，开发者账号免费 |
| **域名** | Cloudflare Registrar（或 Namesilo） | 转移方便，隐私保护 |
| **地图** | Leaflet.js + OpenStreetMap | 免费、GDPR 友好、可离线 |
| **图片优化** | Next.js Image + Cloudflare Images | WebP 自动转换，边缘分发 |

### 6.3 为什么不选 WordPress / Webflow？

| 方案 | 不选理由 |
|------|---------|
| WordPress | 安全维护成本高，SEO 需要插件，页面加载慢 |
| Webflow | 每月费用高，阿拉伯语 RTL 支持弱，定制化受限 |
| Wix / Squarespace | 不利于 SEO，B2B 品牌形象弱 |

**Next.js + Vercel 是目前 ToB 出海官网的最佳性价比组合。**

## 7. 多语言方案

### 7.1 实施方案

使用 **next-intl** 插件，文件结构：

```
/messages
  ├── en.json    # 英文
  └── ar.json    # 阿拉伯语

页面中使用：
t('hero.title')  // 自动根据 locale 返回对应语言
```

### 7.2 RTL 适配要点

| 事项 | 处理方式 |
|------|---------|
| `<html dir="rtl">` | 通过 next-intl middleware 自动设置 |
| Tailwind CSS | 使用 `rtl:` 前缀覆盖样式（如 `rtl:mr-4`） |
| 导航顺序 | 右侧显示 Logo，左侧显示菜单（RTL 模式下自动镜像） |
| 图标方向 | 箭头等图标需单独处理（如 `→` 改为 `←`） |
| 数字格式 | 阿拉伯语使用阿拉伯数字（仍是 0-9） |
| 日期格式 | locale-specific 格式化 |

### 7.3 Hreflang 配置

```html
<link rel="alternate" hreflang="en" href="https://example.com/en/" />
<link rel="alternate" hreflang="ar" href="https://example.com/ar/" />
<link rel="alternate" hreflang="x-default" href="https://example.com/en/" />
```

## 10. 风险与约束

### 10.1 已知风险

| 风险 | 等级 | 应对策略 |
|------|------|---------|
| 阿拉伯语翻译质量不可控 | 🟡 中 | 初期用 AI 翻译快速上线，后期请 native speaker 审校 |
| 中东市场网络访问不稳定 | 🟡 中 | 使用 Cloudflare CDN 优化加载速度 |
| 内容生产周期长 | 🟡 中 | Phase 1 先用占位内容，上线后快速迭代 |
| 多语言 SEO 效果不确定 | 🟡 中 | 参考竞品 hreflang 配置，重点做英文 SEO |
| MVP 功能范围蔓延 | 🔴 高 | 严格执行 MVP 范围，不在 Phase 1/2 增加非必须功能 |

### 10.2 关键约束

- **预算**：优先使用免费/低成本工具（Vercel、Cloudflare、Plausible、Resend）
- **时间**：Phase 1 MVP 目标 2 周上线
- **技术债**：初期可以有一定硬编码内容，但需预留 CMS 扩展空间
